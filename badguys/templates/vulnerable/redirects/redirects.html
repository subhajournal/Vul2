External Redirects

{% endblock %} {% block content %}

It's a pretty common practice to have a page that will redirect to some other URL based on some kind of query parameter... It might be a login page that needs to redirect back to wherever the user was when before they logged in, or it might be a page that gently redirects from an HTTPS to HTTP location to circumvent browser warnings.

But if you don't check the redirect location, it's also a gateway for bad guys to attack your users by sending them along to a malicious location that might install malware, phish for sensitive data, etc.

Our vulnerable codebase has such a page. Attack it and make it go somewhere off site.
