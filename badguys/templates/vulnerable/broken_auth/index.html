Broken Authentication and Session Management

{% endblock %} {% block content %}

Application functions related to authentication and session management are often not implemented correctly, allowing attackers to compromise passwords, keys, session tokens, or exploit other implementation flaws to assume other users' identities.

These kinds of problems often manifest as:

  • session IDs in the query string
  • failure to time out sessions (very bad on shared or public computers)
  • transmitting user login credentials insecurely
  • easily guessable, obtained, or overwritten credentials
